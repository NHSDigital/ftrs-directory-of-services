## Description

Architectural practices supporting high team flow and low coupling.

## Acceptance Criteria

1. GIVEN a service boundary WHEN defined THEN minimal synchronous external dependencies are documented
2. GIVEN a cross-team dependency WHEN required THEN a published API contract with versioning exists
3. GIVEN a quarterly review WHEN performed THEN flow metrics (handoffs and wait time) are analyzed and actions are tracked

## Out of Scope

- Organizational restructuring for product-centric funding

## Implementation Notes

- Evidence: architecture decision records, API contracts, flow metric report
